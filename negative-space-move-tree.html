The move-tree

A text is a path through a tree of possible moves. Each move is a state; each state could have branched otherwise. Drawing the tree makes the road taken, the roads not taken, and the dead-ends all visible at once.

§1A text is a path through a tree

This is the formalism the situation-calculus frame (from Nazmus Saquib) gives us. A move is do(a, s) — doing action a in situation s. Every situation could branch into several next moves; Poss(a, s) decides which are even allowed. The text takes one path; the tree holds all the others. Below, "She Loves You" — the spine is the path the song takes; the branches are what it doesn't; the red boxes are where a precondition fails and the path simply stops.

The reliability branch (a listening insight). M3 — "she's realized he's not the hurting kind" — is drawn with a dashed branch, because it may not be true: it could be her rationalization for what she wants. So M4 ("she loves you") rests on a precondition that is itself unreliable. The song is doubly unreliable: it omits how he feels, and it can't vouch for whether her reappraisal is real. The tree makes that visible as a branch, not a footnote.

§2Two branch-kinds — narrative vs interpretive

Not every branch is the same kind of thing. The first pilot grew the song's tree to 5–9 branches per move — because it conflated two kinds of "otherwise." The fix (the 5th admissibility rule): tag both, never merge them.

[narrative]
an alternative way the story / history could go

she could be angry; the friend could lie; a tradition could spread differently. Plot contingency — scripture is full of these.

[interpretive]
an alternative reading of the same words

"foreknew" = foresight vs fore-love; single vs double predestination. Reading divergence — the fork a doctrine splits on.

Different but not separate. Narrative history — who held power, how a text spread — shapes how the words later get read, so both are kept and rendered distinctly. The move-tree maps how a text is READ (interpretive) laid over how a history could GO (narrative). And the control lives on the interpretive axis: a plain narrative has many [narrative] branches but few [interpretive] forks — it reads one way.
The finding. "She Loves You" is high-narrative, low-interpretive: every branch in the tree above is [narrative] (she's destroyed; keep your pride; her reappraisal is motivated — all plot), and it has no interpretive fork — the song reads one way. Romans forks: "foreknew whom?" is a genuine [interpretive] split (conditional foreseen-faith vs unconditional fore-love), the seam Augustine crossed across 32 years (the trajectory). The instrument distinguishes a text that branches only in plot from one that splits in reading.

§4Toward a real tree — the next build

Right now this tree is the taken path plus a few hand-drawn branches. To make it a real tree, we grow it the way Saquib describes: at each move-state, ask the generation engine for all the ways it could go next — then recurse. Move 1 opens its possibilities; each of those is a new state that opens its own; and so on, until paths either reach the end or stop (a precondition fails). Two shapes are worth building:

  • Open growth: from the first move, branch out fully — every path and every dead-end. The state-space of the song.
  • Fixed endpoints: hold M1 and M8 fixed, and enumerate all the ways M1 could reach M8 — the space of routes between a given start and a given end. (This is the more tractable, and the more revealing for reception: two traditions reaching the same end by different routes.)
The substitute engine already generates the branches; the coding traces the path. What's new is the recursion — growing state by state — and a way to evaluate each generated branch (does it stay in-frame? does its precondition hold?). That evaluation is exactly the reliability work the rest of the lab is built on. On Romans, the tree is where the fork becomes real: the golden chain is one path; single vs. double predestination is a branch.
