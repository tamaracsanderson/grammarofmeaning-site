Method · the coding instrument

How a move gets coded — and how moves become a rhyme-graph.

A passage is read once for its world, then each move inside it is coded on twelve fields. Those fields are a graph: a move is a node, a shared value is a shared node, a rhyme is an edge. This page is the schema, one move fully coded, and the graph the schema produces — every claim labelled.

§1 · The schema12 fields · 2 tiers

The twelve-field grammar — click any field to open its coding key.

The setting is a wrapper, filled once per passage; the move sits inside it, coded per operation. Each field's drawer below is the verbatim codebook — value · what it is · how to tell (the test) · example — the operational definition two coders use to reach the same code.

▢ the setting— the world the move sits in · fill once per passage
◆ the move · the spine— fill per operation
agent → substrate → operation → outcome  (the stock → flow → stock)
◇ the move · qualifiers— fill per operation · the axes that make rhymes computable

The compositional logic: a move = operation-type × voice × target. Named moves fall out — declare = constitute + active; emerge = constitute + stative; reframe = transform + (target: paradigm). No flat sub-types — the axes generate them.

The flow edges — the golden chain. The five verbs of Romans 8:29–30 are five moves on one spine; each move's outcome is the next move's substrate. That's a directed flow inside the graph.

Two edge kinds already visible: the flow edge (within a passage, above) and the rhyme edge (across traditions, in §3). The flow is internal structure; the rhyme is the cross-tradition finding.

§3 · The bridgegrammar = graph

Every field you just coded is a piece of a graph.

Nothing new is invented here. Each grammar field simply is a graph element — so a corpus of coded moves is, already, a network.

the grammar — what you fillthe graph — what it becomes
a coded move (operation)→a node
substrate → operation → outcome→a flow (stock → flow → stock)
agent→an agent node linked to the move
qualifiers (type · voice · mechanism · basis · telos)→node attributes + links to shared value-nodes
shared sub-values (fasting · liberation · by-grace)→shared nodes many moves link to — the convergence substrate
a move-rhyme (same move-type + structure)→an edge between two move-nodes
convergence-degree→node size
The one idea that makes it computable: the shared controlled-vocabularies. Without shared labels, every move is an island. With them, moves connect through shared value-nodes, and convergence becomes "many moves share a neighbour" — exactly what network science computes. Rhyme is computable.

What Saquib's tools then do: once the corpus is a graph, network science finds the structure automatically — shared-neighbour analysis (which moves converge), community detection (clusters of rhyming moves), graph embedding (distance = similarity). Size by convergence-degree and the picture becomes the thesis: the biggest nodes are the moves that recur across the most traditions.
